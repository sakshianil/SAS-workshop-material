# Curated Resources

The sequence below favors authoritative sources before community material.
Standards and regulatory guidance evolve; always confirm the version required
by your study or organization.

## Start with SAS

- [SAS OnDemand for Academics](https://www.sas.com/en_us/software/on-demand-for-academics.html)
  — free cloud access for learners.
- [Open SAS OnDemand](https://welcome.oda.sas.com/)
  — SAS Studio login.
- [SAS documentation](https://documentation.sas.com/)
  — language, procedures, ODS, SQL, macros, and graphics.
- [SAS Communities](https://communities.sas.com/)
  — practical troubleshooting and discussion.

## Learn the standards

- [CDISC standards overview](https://www.cdisc.org/standards)
- [SDTM Implementation Guide](https://www.cdisc.org/standards/foundational/sdtmig)
- [ADaM](https://www.cdisc.org/standards/foundational/adam)
- [Define-XML](https://www.cdisc.org/standards/data-exchange/define-xml)
- [Dataset-JSON](https://www.cdisc.org/standards/data-exchange/dataset-json)
- [CDISC controlled terminology](https://www.cdisc.org/standards/terminology/controlled-terminology)
- [CDISC SDTM/ADaM pilot project](https://github.com/cdisc-org/sdtm-adam-pilot-project)

## A practical reading order

1. Learn basic SAS DATA steps, dates, sorting, SQL, and procedures.
2. Read a high-level clinical-trial and CDISC overview.
3. Complete the synthetic workshop.
4. Review the relevant SDTMIG and ADaM sections for the domains/datasets used.
5. Complete the CDISC pilot mini-workshop.
6. Read the FDA conformance guide and inspect Define-XML/Dataset-JSON examples.
7. Practice explaining lineage and QC evidence aloud, as if responding to a
   reviewer.
